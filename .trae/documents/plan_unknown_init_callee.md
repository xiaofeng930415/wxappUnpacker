# 计划：修复 wuWxml.js 中 Unknown init callee 错误

## 问题分析
用户在反编译过程中遇到了 `Error: Unknown init callee gz$gwx15_1` 错误。
该错误发生在 `wuWxml.js` 的 `analyze` 函数中（约第 245 行）。
原因是对 `gz$gwx` 开头的初始化函数名的正则表达式匹配模式过严，未能匹配带有两位数索引的函数名（如 `gz$gwx15_1`）。
当前的正则 `/(?<=(gz\$gwx\d{0,1}_(XC_){0,1}))[\d|_]+/` 仅允许 `gz$gwx` 后跟 0 或 1 位数字 (`\d{0,1}`)。而 `gz$gwx15_1` 中 `gwx` 后紧跟的是 `15`（两位数字），导致匹配失败。

## 解决方案
修改 `wuWxml.js` 中的正则表达式，允许 `gz$gwx` 后跟任意位数的数字。

## 实施步骤
1.  **修改 `wuWxml.js`**：
    *   定位到第 231 行附近的正则表达式。
    *   将 `\d{0,1}` 修改为 `\d{0,2}`（匹配 0 到 2 位数字）。
    *   修改后的正则应为：`/(?<=(gz\$gwx\d{0,2}_(XC_){0,1}))[\d|_]+/`。

2.  **验证**：
    *   此修改应能正确解析 `gz$gwx15_1`，提取出后缀 `1` 作为 `zMulName`。
    *   修复后，反编译流程应能继续进行，不再抛出异常，从而生成正确的 `.wxml` 文件（而不是降级生成 `.ori.js` 调试文件）。

## 备注
用户提到的 `.ori.js` 文件是 `wuWxml.js` 在捕获异常后自动生成的调试文件。修复上述错误后，该文件可能不再生成（因为不再报错），而是生成目标 `.wxml` 文件。
